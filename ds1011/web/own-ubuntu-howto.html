CS3250 10/11 Distributed Systems: setting up a minimal Ubuntu installation for DS

  • install base Ubuntu (32-bit recommended)
    • on top of Windows: use wubi
    • OR from an installation CD on a real or virtual computer
  • install Java, eg by executing the following in a terminal:
    sudo apt-get install sun-java6-jdk
  • install Eclipse EE
    • open a file manager with administrative privileges, eg by executing the following in a terminal:
      sudo nautilus /usr/local &
    • download and extract Eclipse:
      • download Eclipse IDE for Java EE Developers from here and save it on the desktop
      • drag the archive to /usr/local/ in Nautilus
      • within Nautilus, extract the archive to the folder /usr/local/
    • Create a launch script for eclipse by executing the following in a terminal: 
      sudo touch /usr/local/bin/eeclipse.sh
sudo chmod 755 /usr/local/bin/eeclipse.sh
sudo gedit /usr/local/bin/eeclipse.sh
      and then cutting and pasting the following into the file in gedit: 
      #!/bin/sh
export ECLIPSE_HOME=/usr/local/eclipse
export GDK_NATIVE_WINDOWS=1
$ECLIPSE_HOME/eclipse $*
      and then saving and closing gedit.
  • install Mantaray
    • Download mantaray_java_2.0.1_bin.zip and extract it to /usr/local/ in the same way as eclipse.
    • Rename the extracted folder in /usr/local/ as follows:
      • mantaray_2.0.1_bin → mantaray
    • Disable IPv6 networking support in the kernel:
      • edit the file /etc/default/grub, eg by executing:
        sudo gedit /etc/default/grub
      • find the line that says GRUB_CMDLINE_LINUX=""
      • change the line to: GRUB_CMDLINE_LINUX="ipv6.disable=1"
      • save the file, close the editor and execute the following:
        sudo update-grub
      • this change will take effect after the next reboot as is only needed when using Mantaray
  • install Tomcat
    • Download Tomcat archive , and extract it to /usr/local/ in the same way as eclipse.
    • Rename the extracted folder in /usr/local/ as follows:
      • apache-tomcat-6.0.29 → tomcat
  • install Jersey
    • In Nautilus, create folder jersey in /usr/local/.
    • Download the following archives and move them into /usr/local/jersey:
      jersey-bundle-1.3.jar, jsr311-api-1.1.jar, asm-3.1.jar
  • You should now be able to do all DS practical tasks on your minimal Ubuntu system with minimal changes.
